# 664背景

“664 总线” 通常指 ARINC 664 总线，也被称为 AFDX（Avionics Full-Duplex Switched Ethernet），即航空电子全双工交换式以太网，是一种基于以太网的通信标准。以下是关于它的详细介绍：

## **发展背景**：

随着航空电子技术的发展，早期的 ARINC 429 等单向、串行数据总线协议已无法满足现代航空电子设备对数据传输速率和通信容量的要求，ARINC 664 应运而生，它由国际航空电子协会（ARINC）制定，旨在为航空电子系统提供更高效的通信解决方案。

## 技术特点

- **全双工通信**：基于以太网 100Base-TX（100Mbps），采用全双工通信模式，数据可以在任意时刻向两个方向进行传输，大大提升了通信效率。
- **星形拓扑结构**：其网络主要元素包括端系统、交换机和链路，端系统通过链路连接到交换机，交换机之间采用星型拓扑结构相连，保证数据可以在航空器内部的各个系统之间高效传输。
- **虚拟链路技术**：引入虚拟链路（VL）的概念，它定义了数据包必须遵循的路径，具有保证带宽、最大延迟和流量排他性等特点，确保关键系统能以可预测和安全的方式接收信息。
- **高可靠性**：支持冗余通信，通过两个独立通道传输数据，同时具备流量控制、拥塞管理和服务质量（QoS）特性，保证了关键通信的优先级和带宽分配，提高了数据传输的可靠性。



## 664虚拟链路：

ARINC 664 总线的虚拟链路（VL）是一种逻辑上的点到点连接，它为网络中的每个数据流建立了一条预定的 “通道”。每个虚拟链路都有唯一的标识，即虚拟链路标识符（VLI）。在数据传输过程中，端系统将数据包发送到交换机，交换机根据数据包中的虚拟链路号，按照预先配置好的路由规则，将数据包准确地传输到目标端系统。

虚拟链路在系统设计阶段就已确定其属性，包括带宽、周期性、延迟等参数。这些参数在运行时通过严格的流量控制和调度策略得以执行，确保数据能够在预定的路径上以保证的带宽和确定的延迟进行传输，有效避免了网络拥堵，并确保关键数据流的传输质量。例如，在航空电子系统中，飞行控制计算机的数据可以通过特定的虚拟链路，按照预定的路径和带宽要求，快速、可靠地传输到相关的执行机构，保证飞行控制的实时性和准确性。
